The Code Structure of Auto Mech¶

The overall workflow of Auto Mech is driven by moldriver, whose overarching task is to invoke electronic structure calculations through the elstruct interface and feed the results into MESS for calculating partition functions and kinetic rates.

Data generated during the workflow is written to a file system database using the autofile module, which also provides functions for retrieving previously stored data.

The aforementioned codes are supported by several helper libraries:

  • automol provides routines for manipulating and interconverting between molecular descriptors (Cartesian geometry, z-matrix, molecular graph, InChI, SMILES, molecular formula)

  • interfaces contains a hodge-podge of interaces to various programs

  • autoparse is a low-level parsing library employed by the various I/O modules
